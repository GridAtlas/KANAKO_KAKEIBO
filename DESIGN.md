## データフロー

### 取引登録

```
Add page
  → TransactionForm.submit
    → useBudgetStore.addTransaction(input)
      → transactions.push({ ...input, id, createdAt })
    → toast + router.push("/")
    → persist automatically triggered
```

### 月次サマリの計算

```
Home renders
  → useBudgetStore(state => state.getMonthlyTotals(selectedMonth))
    → filter transactions by month
    → group by category.type
    → sum expense / income
    → return { expense, income, balance }
```

### 固定費適用

```
User opens app / clicks "今月の固定費を適用"
  → useBudgetStore.applyFixedCostsForMonth("2026-06")
    → for each active FixedCost:
      → if already applied for this month: skip
      → construct Transaction (date = YYYY-MM-{clamped payment_day})
      → addTransaction({ ..., isAutoGenerated: true, fixedCostId })
      → mark appliedFixedCostMonths[fixedCostId].push("2026-06")
```

## 重要ロジック詳細

### `applyFixedCostsForMonth(monthKey)`

```ts
function applyFixedCostsForMonth(monthKey: string) {
  const [year, month] = monthKey.split("-").map(Number);
  const daysInMonth = new Date(year, month, 0).getDate();

  for (const fc of fixedCosts.filter(f => f.active)) {
    const alreadyApplied = appliedFixedCostMonths[fc.id]?.includes(monthKey);
    if (alreadyApplied) continue;

    const day = Math.min(fc.paymentDay, daysInMonth); // 2/30 → 2/末
    const date = `${monthKey}-${String(day).padStart(2, "0")}`;

    addTransaction({
      categoryId: fc.categoryId,
      amount: fc.amount,
      date,
      memo: fc.label ?? "固定費",
      isAutoGenerated: true,
      fixedCostId: fc.id,
    });

    appliedFixedCostMonths[fc.id] = [...(appliedFixedCostMonths[fc.id] ?? []), monthKey];
  }
}
```

**Idempotency**: 同月に 2 回呼ばれても重複計上しない。

### 削除時の関連 Transaction 扱い（Phase 1 で決定）

現状: **removeCategory はカテゴリだけ消し、Transaction の categoryId は残す**（display 時に "削除済カテゴリ" と表示）。将来的に UX 決定次第で移す。

### 収支の集計

```ts
function getMonthlyTotals(monthKey: string) {
  const txs = getMonthlyTransactions(monthKey);
  let expense = 0, income = 0;
  for (const t of txs) {
    const cat = categories.find(c => c.id === t.categoryId);
    if (cat?.type === "income") income += t.amount;
    else expense += t.amount; // 削除済カテゴリは支出扱い
  }
  return { expense, income, balance: income - expense };
}
```

### 円グラフ用のカテゴリ集計

```ts
function getMonthlyByCategory(monthKey: string) {
  const txs = getMonthlyTransactions(monthKey);
  const map = new Map<string, number>();
  for (const t of txs) {
    map.set(t.categoryId, (map.get(t.categoryId) ?? 0) + t.amount);
  }
  return Array.from(map.entries())
    .map(([categoryId, total]) => ({
      category: categories.find(c => c.id === categoryId) ?? DELETED_CATEGORY,
      total,
    }))
    .filter(x => x.category.type === "expense")
    .sort((a, b) => b.total - a.total);
}
```

## ステート遷移（Category / Transaction）

```
[Category]
  new → active → (削除) → 削除済 (orphan、Transaction は categoryId で参照残る)

[Transaction]
  new → committed
       ↓ (edit)
       committed
       ↓ (remove)
       gone

[FixedCost]
  new → active ↔ inactive → deleted
                    ↓
                    生成した Transaction は残る
```

## モバイル UI 詳細

### Home タイトル / 分析セクション

- Home 先頭に `TitleHero` を表示し、`public/images/kanako-title.jpg` を背景画像として使う。表示タイトルは「かなこの家計簿VER0.2」。
- `MonthlyInsights` は `selectedMonth` を基準に直近 6 か月を `shiftMonthKey` で作り、`getMonthlyTotals(monthKey)` から収入 / 支出 / 収支を算出する。
- 貯金率は `balance / income * 100`。収入が 0 の月は `--` 表示として、ゼロ除算しない。
- 月次推移グラフは recharts の `BarChart`。ピンクを支出、グリーンを収入に固定し、データは localStorage 由来の store だけから読む。

## スタイル方針

- Tailwind utility class 直接
- 色設計:
  - Primary: `pink-500` / 背景: `pink-50` とクリーム系グラデーション
  - 支出: `text-rose-600 bg-rose-50 border-rose-200`
  - 収入: `text-emerald-600 bg-emerald-50 border-emerald-200`
  - 固定費バッジ: `text-amber-700 bg-amber-100`
- フォント: システムフォント（layout.tsx で Noto Sans JP フォールバック）
- 数値: `font-mono tabular-nums`（金額）、`font-semibold`

## セキュリティ・プライバシー

- Phase 1〜2: **サーバに一切送らない**、localStorage のみ
- Phase 3: Firebase 導入時に暗号化なし（Firestore 依存）
- 個人情報は扱わない（名前 = "かなこ" 等はローカル）
- レシート画像は Phase 3 で Firebase Storage / user 別 bucket

## パフォーマンス

- Zustand selector で subscribe を最小化
- 月次集計は selector で `useMemo` 相当（selector 内でメモ化）
- Transaction が数千件を超えたら indexeddb 移行検討（Phase 2 後半）

## 将来の拡張ポイント

- **予算機能**: Category に `monthlyBudget?: number` を追加、Home で消化率を表示
- **タグ**: カテゴリと直交する tag システム（"必要", "浪費" 等）
- **繰り返し支出**: 固定費とは別の「隔週」「毎週金曜」等の repeat
- **サブアカウント**: 「かなこ」「かずき」等の分離（Phase 4）

## 制約

- iOS Safari の date input UI はカスタマイズ困難 → 早期に react-day-picker 等を導入検討
- localStorage 5-10MB 制限 → 数年分のデータで到達する可能性
- SSR で store のデータを描画すると hydration mismatch → **client only page** にする（`"use client"`）
