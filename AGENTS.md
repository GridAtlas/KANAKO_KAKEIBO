Codex（および Claude 等の AI）が KANAKO_KAKEIBO を拡張する際の指針。**初回作業前に本ファイルと `DESIGN.md` を必ず読むこと**。

## 1. 設計哲学

### 1.1 「入力を最小・可視化を最大」

家計簿の 90% は「金額を入力する」の繰り返し。**タップ数を最小化**することが最重要。

- 金額入力は `inputMode="decimal"` でテンキー起動
- カテゴリ選択は 1 タップ（グリッドから直接）
- 日付は「今日」がデフォルト
- Add タブに入ってから登録完了まで **3 タップ以内**を目安

### 1.2 モバイルファースト

- **ボトム 3 タブ + 中央 FAB**: Home / History / Settings + `+`
- タップターゲット最小 44px
- iOS 自動 zoom 防止: 全 input は 16px 以上
- スワイプ操作は当面**なし**（誤操作リスク > 便利さ）

### 1.3 個人アプリ・オフライン優先

- Phase 1〜2 は**アカウント不要・オフラインで全機能動作**
- データは localStorage のみ
- Firebase 同期は Phase 3 でオプトイン

### 1.4 状態は Zustand 1 本

複数 store に分割しない。`useBudgetStore` が正本。persist middleware で localStorage 永続化。

## 3. データモデル

`src/types/index.ts` に定義。仕様書に準拠しつつ、シングルユーザー MVP なので `user_id` は省略。

```ts
export type CategoryType = "expense" | "income";

export interface Category {
  id: string;              // uuid
  name: string;            // "食費", "サウナ代" 等
  icon: string;            // Lucide アイコン名 "Utensils" 等
  color: string;           // Tailwind color: "red" | "blue" | ... （chart 用）
  type: CategoryType;      // 支出 or 収入
  isFixedCost: boolean;    // 固定費カテゴリか
  sortOrder: number;       // 表示順
  createdAt: number;
}

export interface Transaction {
  id: string;
  categoryId: string;
  amount: number;          // 円（整数）
  date: string;            // "YYYY-MM-DD"（ローカル時刻）
  memo?: string;
  receiptImageUrl?: string; // Phase 3
  isAutoGenerated: boolean; // 固定費から自動生成された？
  fixedCostId?: string;    // isAutoGenerated=true のとき参照
  createdAt: number;
}

export interface FixedCost {
  id: string;
  categoryId: string;
  amount: number;
  paymentDay: number;      // 1-31
  label?: string;          // "Netflix", "家賃" などの補助ラベル
  active: boolean;         // 一時停止対応
  createdAt: number;
}
```

### 不変条件

- `Category.id` は UUID、名前 unique 保証はしない（重複 OK、ユーザー自由）
- `Transaction.date` は "YYYY-MM-DD"（ローカル時刻ベース、UTC 変換しない）
- `Transaction.amount` は正の整数（円）。**符号で収入/支出を区別しない**。区別は `Category.type` から
- `FixedCost.paymentDay` が該当月に存在しない場合（2/30 等）はその月の月末に丸める（`lib/fixedCosts.ts`）
- `Transaction.isAutoGenerated === true` の Transaction は編集で元 FixedCost を切断してもよい（`fixedCostId` を null に）

## 4. Zustand store

### 4.1 State

```ts
interface BudgetStore {
  categories: Category[];
  transactions: Transaction[];
  fixedCosts: FixedCost[];
  selectedMonth: string; // "YYYY-MM"
  // 固定費自動生成の適用済み月キー（重複計上防止）
  appliedFixedCostMonths: Record<string /* fixedCostId */, string[]>;
}
```

### 4.2 Actions（実装済）

```ts
// Category
addCategory(input): Category
updateCategory(id, patch)
removeCategory(id)  // 関連 Transaction は保持、categoryId を "" にはしない（DELETED sentinel を検討中）
reorderCategories(ids)  // sortOrder を上書き

// Transaction
addTransaction(input): Transaction
updateTransaction(id, patch)
removeTransaction(id)

// FixedCost
addFixedCost(input): FixedCost
updateFixedCost(id, patch)
removeFixedCost(id)

// Month navigation
setSelectedMonth(monthKey: string)  // "YYYY-MM"

// Fixed cost application（ユーザー操作 or 起動時）
applyFixedCostsForMonth(monthKey: string)
  // その月の appliedFixedCostMonths に含まれない FixedCost を Transaction 化
  // 重複計上しない

// Selectors
getMonthlyTransactions(monthKey)
getMonthlyTotals(monthKey): { expense, income, balance }
getMonthlyByCategory(monthKey): Array<{ category, total }>
```

### 4.3 persist middleware

`kanako_kakeibo:v1` キーで localStorage に保存。`version: 1`。将来スキーマ変更したら `migrate` fn を書く。

## 5. 画面フロー

### 5.1 タブ構成

```
┌──────────────────────────────┐
│                              │
│   Content Area (ページごと)  │
│                              │
├──────────────────────────────┤
│  🏠 Home  📋 History  ⚙️ Set │  ← Bottom Nav
│         [＋ FAB]              │  ← 中央に浮かぶ大 FAB
└──────────────────────────────┘
```

- **Home** (`/`): ダッシュボード。月次 total、円グラフ（Phase 2）、直近履歴 5 件
- **History** (`/history`): 全取引一覧、カテゴリ・月でフィルタ
- **Settings** (`/settings`): 設定ハブ → カテゴリ管理 / 固定費管理
- **FAB (+)**: `/add` に遷移

### 5.2 登録フロー

`/add`:
1. 金額入力（テンキー）
2. カテゴリ選択（グリッド、初期は最近使ったカテゴリ）
3. 日付選択（デフォルト今日）
4. メモ（任意）
5. 保存 → Home に戻る（トースト表示）

## 6. Phase 2 で追加する円グラフ

`ExpensePieChart.tsx` は recharts の `PieChart` を使う。

- データ: `getMonthlyByCategory(selectedMonth).filter(x => x.category.type === "expense")`
- タップで該当カテゴリの詳細（`/history?category=xxx&month=xxx`）に遷移
- 色は `Category.color`（Tailwind カラー名から HEX にマップ、`lib/icons.ts` 隣に `lib/colors.ts` を作る想定）

---

## 7. Phase 3 で追加する OCR

- カメラ起動: `<input type="file" accept="image/*" capture="environment">`
- 画像を Cloud Vision API に POST
- レスポンスから日付・合計金額を抽出（正規表現ベース、後々改善）
- フォームに自動補完

**セキュリティ**: API キーは server-side で保持、Next.js API Route (`/api/ocr`) 経由で呼ぶ。クライアントに直接埋め込まない。

## 8. Firebase 同期（Phase 3）

- 認証は Google Sign-in
- Firestore にコレクション `users/{uid}/{categories,transactions,fixedCosts}`
- 起動時に localStorage と Firestore を merge（`updatedAt` 大きい方を採用）
- オフラインでも動作継続、次回オンライン時に同期

## 10. お作法

### 10.1 新機能追加時

- ROADMAP.md の対応 Phase を確認・更新
- 大きな設計変更は DESIGN.md に追記
- Category/Transaction/FixedCost の型を変更するときは persist middleware の `version` を上げて `migrate` を書く

### 10.2 コミット

- 1 コミット 1 トピック
- 日本語 OK、subject 50 文字以内
- 破壊的変更は commit body で明記

### 10.3 テスト（Phase 2 で着手）

- vitest + happy-dom
- 対象:
  - `lib/date.ts`, `lib/money.ts`, `lib/fixedCosts.ts`
  - store の actions（特に `applyFixedCostsForMonth` の重複防止）
  - `getMonthlyByCategory` の集計正しさ

### 10.4 ハマりやすいポイント

- iOS Safari の date input はデフォルトで美観が悪い → カスタム date picker 検討（Phase 2）
- `Intl.NumberFormat("ja-JP", { style: "currency", currency: "JPY" })` は "￥1,234" になる。「¥」記号を先頭にしたい場合は自作
- 月キーは "YYYY-MM"。文字列比較でソート可能
- localStorage の容量制限（5-10MB）に近づく前に export/import 機能を（Phase 2 で）
- Zustand persist は SSR で hydration mismatch を起こしやすい → `useEffect` で client-only レンダに
